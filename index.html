The name Parcela comes from the Spanish (the mother tongue of one of the authors) for parcel in the sense of "a distinct, continuous portion or tract of land". A parcel in Parcela represents a small portion of a web page.

Parcela is a community-project. We develop basic commonjs-modules which can be used on both server and client (isomorphic). We demand this feature to be able to do serverside developing and rendering. All modules are highly tested, well documented and high performant. Among the modules are:

  • Virtual DOM (using Parcels) work in progress
  • Promised IO
  • Rich Event system
  • I-tags (isomorphic custom elements) work in progress
Get Started
Creating a webapplication or server-application is just a matter of:
  • Setup package.json for your project
  • For browserenvironments: create a buildfile
  • Create your code reqquireing the modules
You are free to mix Parcela modules with other npm-modules.

Example package.json

{% highlight json %} { "name": "my-project", "version": "0.0.1", "dependencies": { "io": "Parcela/io", "io-transfer": "Parcela/io-transfer" } } {% endhighlight %}

With nodejs:

{% highlight js %} var IO = require('io'), IOtransfer = require('io-transfer'); // delivers the method 'get' IOtransfer.mergeInto(IO); IO.get('http://somedomain.com/getInfo?q=something').then( function(response) { // `response` holds the remotedata } ); {% endhighlight %}
