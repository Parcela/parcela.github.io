The name Parcela comes from the Spanish for parcel in the sense of "a distinct, continuous portion or tract of land". A parcel in Parcela represents a small portion of a web page. The logo is inspired by the colorful fields of tulips in The Netherlands.

Parcela is a community-project. We develop basic commonjs-modules which can be used on both server and client (isomorphic). We demand this feature to be able to do serverside developing and rendering. All modules are highly tested, well documented and highly performant.

Parcela is designed with the following goals in mind:

  • Completely modular. All different parts can be mixed and matched to suit any production site
  • A standard distribution is provided to make it easy to get started
  • Rendering is via a Virtual DOM that tries to minimize the costly access to the browser's DOM, to make fast and responsive updates.
  • Client side routing designed for single-page applications.
  • Isomorphic. All modules must be able to run both on the client and the server.
  • Unidirectional flow of data: the screen is a faithful representation of the state of the data in the application, conveniently transformed for the user.
  • Dynamic module loading: only a minimal bundle has to be loaded initially, extra modules can be loaded on-demand.
  • Rich set of ready-made components.

Getting Started

Parcela provides a standard distribution which aggregates the various modules that make up Parcela into one convenient Browserify bundle to be used in clients and an npm package to be used server-side. The standard distribution is rooted in the parcela module.

To start using Parcela, the following resources are available:

  • Client side Browserify bundle: parcela.js
  • Client side, minified Browserify bundle: parcela-min.js
  • Npm `package.json` file to install via `npm install`: package.json

Creating a client-side web application is just a matter of loading the client side bundle and then using require to get a reference to the root namespace of the standard distribution.
